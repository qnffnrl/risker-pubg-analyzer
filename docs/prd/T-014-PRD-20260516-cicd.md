# T-014 PRD — CI/CD (GitHub Actions → risker.co.kr)

> 상태: 📋 대기 | 생성일: 2026-05-16 | 유형: infra

## 목표

GitHub Actions로 main 브랜치 푸시 시 Mac Mini 서버(risker.co.kr)에 자동 배포한다.
앱 접속 URL: `http://risker.co.kr/risker-pubg-analyzer`

## 배포 아키텍처

```
GitHub main push
  → GitHub Actions (CI: 빌드 + 테스트)
  → SSH → risker.co.kr Mac Mini
      → git pull
      → docker-compose -f docker-compose.prod.yml up -d --build
      → 헬스체크 통과 확인
```

### 서버 구성 (Mac Mini @ risker.co.kr)
- Docker Desktop for Mac (또는 Docker CE) 설치 필요
- **Caddy** 리버스 프록시: Let's Encrypt HTTPS 자동 처리 (이미 운영 중)
- PostgreSQL + Redis: Docker 컨테이너
- 앱: Docker Compose (web:8080, api:8081, worker Bull Board:8082)
- 앱 URL: `https://risker.co.kr/risker-pubg-analyzer` (HTTPS)

### Caddy 설정 (`/etc/caddy/Caddyfile` 또는 기존 Caddyfile에 추가)
```caddyfile
risker.co.kr {
    # 기존 설정 유지 ...

    # PUBG Analyzer — web (Next.js)
    handle /risker-pubg-analyzer/* {
        reverse_proxy localhost:8080
    }

    # PUBG Analyzer — API (Hono)
    handle /risker-pubg-analyzer/api/* {
        uri strip_prefix /risker-pubg-analyzer
        reverse_proxy localhost:8081
    }
}
```
> Caddy가 Let's Encrypt 인증서를 자동으로 발급/갱신하므로 SSL 설정 별도 불필요.

## 배포 격리 원칙

**호스트 시스템에 영향 없는 완전 컨테이너 격리:**
- 앱 코드, DB, Redis, Worker — 모두 Docker 컨테이너로만 실행
- 호스트(Mac Mini)에는 Docker + Nginx + Git만 설치
- 모든 포트는 Docker 내부 네트워크 — Nginx가 외부 노출 담당
- 컨테이너 재시작/업데이트가 호스트 시스템에 영향 없음

## GitHub Actions 워크플로우

### `.github/workflows/ci.yml` (PR/브랜치 CI)
```yaml
on: [push, pull_request]
jobs:
  ci:
    - pnpm install
    - turbo lint
    - turbo typecheck
    - turbo test
    - turbo build
```

### `.github/workflows/deploy.yml` (main → 자동 배포)
```yaml
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to Mac Mini
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.SERVER_HOST }}        # risker.co.kr
          username: ${{ secrets.SERVER_USER }}    # SSH 유저
          key: ${{ secrets.SERVER_SSH_KEY }}      # SSH 개인키
          port: 2222                              # 서버 SSH 포트
          script: |
            cd /opt/risker-pubg-analyzer
            git pull origin main
            docker compose -f docker-compose.prod.yml up -d --build --remove-orphans
            docker compose -f docker-compose.prod.yml ps
            
      - name: Health Check
        run: |
          sleep 30
          curl -f http://risker.co.kr/risker-pubg-analyzer || exit 1
```

## GitHub Secrets (서버에서 설정 필요)

| Secret 이름 | 값 |
|------------|-----|
| `SERVER_HOST` | `risker.co.kr` |
| `SERVER_USER` | SSH 접속 유저명 |
| `SERVER_SSH_KEY` | SSH 개인키 (PEM) |
| `ANTHROPIC_API_KEY` | Claude API 키 |
| `DB_PASSWORD` | PostgreSQL 비밀번호 |
| `ADMIN_PASSWORD` | 관리자 페이지 비밀번호 (T-018) |

> **PUBG_API_KEY**: 서버의 `.env.prod` 파일에 직접 설정 (GitHub Secrets 불필요)
