# T-008 PRD — 플레이어 검색 페이지

> 상태: 📋 대기 | 생성일: 2026-05-16 | 유형: feat

## 목표

사용자가 닉네임으로 PUBG 플레이어를 검색하는 랜딩/홈 페이지를 구현한다.
최근 검색 + 즐겨찾기를 로컬 스토리지에서 불러와 빠른 재접근을 지원한다.

## 페이지 구성

### URL
`https://pubg.risker.co.kr` (홈, 검색)

### 레이아웃 (데스크톱)
```
[Header: 로고 + 검색바]
[Hero: 대형 타이틀 + 메인 검색 입력]
[즐겨찾기 플레이어 (로컬 스토리지)]
[최근 검색 플레이어 (로컬 스토리지)]
[하단: 통계 (총 분석 플레이어 수, 총 분석 매치 수 — API에서 조회)]
```

### 레이아웃 (모바일)
```
[Header: 로고 + 햄버거]
[검색 입력 (전체 너비)]
[최근 검색 / 즐겨찾기 탭]
[통계 카드]
[BottomNav]
```

## 검색 컴포넌트

### 입력 필드
- 닉네임 텍스트 입력 (최소 3자)
- 플랫폼 선택: `Steam` / `Kakao` / `Console` (토글 버튼)
- 검색 버튼 + Enter 키 지원
- 자동완성: 로컬 스토리지의 최근 검색 기반 드롭다운

### 검색 결과 UX
1. 검색 버튼 클릭 → API 호출 → 로딩 스피너
2. 캐시 히트: 즉시 분석 페이지로 이동
3. 캐시 미스: "데이터 수집 중..." 프로그레스 바 (잡 폴링, 500ms 간격)
4. 완료: 자동으로 플레이어 분석 페이지(`/players/{id}`)로 이동
5. 에러: 토스트 메시지 (플레이어 없음, 네트워크 오류 등)

## Hero 섹션 디자인 (openclaw.ai 참고)

- 배경: 어두운 그라데이션 (군사적 분위기)
- 타이틀: "당신의 배그 DNA를 분석하세요" (대형 타이포그래피)
- 서브타이틀: "킬, 생존, 포지셔닝, 팀플 — 4가지 성향으로 보는 플레이 스타일"
- 입력 아래: 플레이어 통계 숫자 (분석된 플레이어 N명, 분석된 매치 N판)
- 애니메이션: Framer Motion 진입 효과
