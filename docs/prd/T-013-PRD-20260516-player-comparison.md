# T-013 PRD — 플레이어 비교 기능

> 상태: 📋 대기 | 생성일: 2026-05-16 | 유형: feat

## 목표

두 플레이어의 플레이 스타일을 나란히 비교하는 기능을 구현한다.

## 페이지 구성

### URL
`http://risker.co.kr/risker-pubg-analyzer/compare?a={pubgId1}&b={pubgId2}`
- URL 파라미터로 공유 가능

### 레이아웃

**진입 방식**
1. 플레이어 프로필 페이지의 "비교하기" 버튼
2. 직접 URL 입력
3. 홈 페이지 비교 버튼 (검색 2개)

**비교 화면 (데스크톱)**
```
[플레이어 A 헤더]          [VS]          [플레이어 B 헤더]
     닉네임 + 플랫폼                         닉네임 + 플랫폼

[오버레이 레이더 차트 — 두 플레이어 겹쳐 표시]

[공격성]  A: 72 ████████░░  B: 45 █████░░░░░
[생존형]  A: 38 ████░░░░░░  B: 81 █████████░
[포지셔닝] A: 61 ██████░░░░  B: 57 ██████░░░░
[팀플레이] A: 55 ██████░░░░  B: 78 ████████░░

[상세 지표 비교 테이블]
항목          플레이어A    플레이어B    승자
킬/매치        3.2          2.1         A ↑
데미지/매치    285          198         A ↑
헤드샷율       31%          28%         A ↑
평균 순위      8.3          6.1         B ↑
탑10율         45%          58%         B ↑
승률           8%           12%         B ↑
...
```

**레이더 차트 (오버레이)**
- 플레이어 A: 사이언 컬러, 반투명 fill
- 플레이어 B: 앰버 컬러, 반투명 fill
- 범례 + 각 축 점수

**모바일**
- 탭으로 A/B 전환
- 차트는 겹쳐서 표시 (작게)
- 비교 바는 세로로 나열

## 데이터 페칭

```typescript
// 두 플레이어 분석 결과 병렬 조회
const [analysisA, analysisB] = await Promise.all([
  api.players.getAnalysis(pubgIdA),
  api.players.getAnalysis(pubgIdB),
])
```

- 한 명이 분석 안 된 경우: 분석 트리거 + 폴링

## 비교 페이지 진입 플로우

1. 프로필 페이지 → "비교하기" 클릭
2. 비교 페이지로 이동 (플레이어 A 고정)
3. 플레이어 B 검색 입력
4. 비교 결과 표시

## 완료 조건

- [ ] 오버레이 레이더 차트 (두 플레이어 겹쳐 표시)
- [ ] 성향 점수 바 비교 (4개)
- [ ] 상세 지표 비교 테이블 (승자 표시)
- [ ] URL 파라미터로 공유 가능
- [ ] 플레이어 B 검색 입력 기능
- [ ] 모바일 탭 전환 레이아웃
- [ ] 분석 미완료 플레이어 자동 분석 트리거
