# T-025 PRD — Refresh UX 개선 (랭크/분석 캐시 갱신 피드백)

**작성일**: 2026-05-17  
**상태**: 대기 (승인 전)  
**유형**: feat

---

## 배경

현재 플레이어를 검색하면 분석 캐시(play_style_analyses)가 유효한 동안 API가 새 데이터를 반환하지 않는다. ranked stats 같은 신규 기능이 추가되어 worker가 데이터를 수집했더라도, 캐시 만료 전까지는 UI에서 확인 불가. 사용자는 캐시 만료 시점을 알 수 없어 새로고침을 반복해야 한다.

## 문제

1. **새로고침 버튼 → 즉각 피드백 없음**: `/refresh` API 호출 후 worker 수집이 끝날 때까지 UI 변화 없음
2. **캐시 만료 전 신규 기능 미표시**: ranked stats처럼 새로 추가된 데이터는 캐시 만료 후에야 보임
3. **수집 완료 시점 불명확**: "언제 새로고침하면 되나"를 사용자가 모름

## 변경 범위

### 1. `apps/api` — `/refresh` 응답 개선
- 현재: `{ jobId, message }` 반환
- 변경: `{ jobId, message }` 동일하나, 이후 `/api/v1/jobs/:jobId` 폴링으로 완료 여부 확인 가능 (기존 jobs 라우트 활용)

### 2. `apps/web` — 새로고침 버튼 UX
- 새로고침 버튼 클릭 시:
  1. `POST /refresh` 호출 → jobId 획득
  2. "수집 중..." 스피너 표시 (버튼 비활성화)
  3. `/api/v1/jobs/:jobId` 2초 간격 폴링
  4. job 완료(completed) → 페이지 자동 리로드
  5. 실패 또는 30초 타임아웃 → "수집 실패, 잠시 후 다시 시도" 토스트
- 새로고침 버튼 위치: PlayerHeader 우측 상단 (기존 버튼 동작 교체)

### 3. ranked stats 미수집 시 안내
- 랭크 탭 진입 시 ranked_stats 없으면 현재 "랭크 데이터가 없습니다" 메시지에 "새로고침으로 수집할 수 있습니다" 안내 추가

## 완료 조건

- [ ] 새로고침 버튼 클릭 → 수집 중 스피너 표시
- [ ] job 완료 감지 → 자동 페이지 리로드
- [ ] 30초 내 미완료 시 타임아웃 메시지
- [ ] 랭크 탭 빈 상태 메시지에 새로고침 안내 추가
