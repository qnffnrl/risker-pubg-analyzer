# T-020 PRD — 시즌 & 모드 필터 (신규 기능)

> 상태: 📋 대기 | 생성일: 2026-05-16 | 유형: feat

## 목표

플레이어 분석 시 시즌, 게임 모드, 플랫폼을 필터링해 더 세밀한 분석을 제공한다.
"솔로만 봤을 때 더 강한지", "스쿼드 전문가인지" 등을 파악할 수 있다.

## 필터 옵션

### 게임 모드
| 값 | 표시명 |
|----|--------|
| `all` | 전체 |
| `squad` | 스쿼드 |
| `squad-fpp` | 스쿼드 FPP |
| `duo` | 듀오 |
| `duo-fpp` | 듀오 FPP |
| `solo` | 솔로 |
| `solo-fpp` | 솔로 FPP |

### 맵
| 값 | 표시명 |
|----|--------|
| `all` | 전체 |
| `Baltic_Main` | 에란겔 |
| `Desert_Main` | 미라마 |
| `Savage_Main` | 사녹 |
| `Tiger_Main` | 태이고 |
| `Kiki_Main` | 데스턴 |
| 등 | |

### 기간
- 최근 10게임
- 최근 20게임 (기본)
- 최근 50게임

## UI 구성

### 필터 바 (플레이어 프로필 + 분석 대시보드 상단)
```
[모드: 전체 ▼] [맵: 전체 ▼] [기간: 최근 20게임 ▼]
```
- 드롭다운 선택
- 필터 변경 시 즉시 데이터 재계산 (클라이언트 사이드 필터링)
- URL 쿼리스트링 반영 (`?mode=squad&map=all&period=20`) → 공유 가능

### 모드별 빠른 비교 카드 (선택적 UI)
```
[스쿼드] 킬: 3.8 / 순위: 7.2 / 승률: 9%
[솔로]   킬: 2.1 / 순위: 12.4 / 승률: 3%
→ "스쿼드 특화 플레이어"
```

## 구현 방식

### 클라이언트 사이드 필터링 (기본)
- 이미 로드된 20개 매치를 JavaScript로 필터링
- 빠른 반응성

### 서버 사이드 필터링 (기간 변경 시)
- 기간 변경 → API 재요청 (`?limit=50`)

### 분석 재계산
- 필터 적용된 매치셋으로 4대 성향 점수 재계산
- 클라이언트에서 분석 엔진의 순수 함수(`packages/shared`)를 재사용

```typescript
// packages/shared에서 export
export function calculatePlayStyle(matches: PlayerMatchStats[]): PlayStyleScores
```
